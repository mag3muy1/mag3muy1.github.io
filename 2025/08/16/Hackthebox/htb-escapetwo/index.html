  3. access the smb shares via smbclient

  4. unzip both the xlsx

  5. gather the information and get the following results

  6. using impacket-mssqlclient, establish connection with the sql server hosting on 1433

  7. using the command from https://github.com/noraj/the-hacking-trove/blob/master/docs/Cheat%20Sheets/mssql.md, activate xp_cmdshell for further enumeration

  8. after some enumeration, got this configuration file

interesting finding:

we can deduct that possibly ryan who has administrator privileges created a service named sql_svc, so its possible for the password to be reuse. What this means is that the password in sql-Configuration.INI might be password for ryan as well

  1. with that, evil-winrm into ryan with the password from sql-Configuration.INI

  1. Until here, we can advance to bloodhound and notice user Ryan have ‘WriteOwner’ towards CA_SVC

  2. to exploit ‘WriteOwner’, refer to the Linux Abuse by the bloodhound modify the command.

  3. after gaining full control over CA_SVC, we can perform certificate check via certipy to verify the vulnerability in the templates

  4. notice ESC4 exist, refer to a website https://redfoxsec.com/blog/exploiting-weak-acls-on-active-directory-certificate-templates-esc4/

to verify the updated template consists of ESC1 to 4

  1. then, exploit it via method of exploiting ESC1

  2. using certipy, authenticate using the certificate [administrator.pfx] and get the TGT Ticket and NTLM hash

  3. after that, log in to administrator via impacket-psexec and pwn
