nmap

Port 80 is hosting a website named Best Cat Competition with .git available, which opens up possibility to use git-dumper.

Visiting port 80

Available phps:

  1. vote.php [“visit the website returns: Voting is currently closed”]
  2. contest.php [“requires us to register”]
  3. winners.php [“displays cats that win the competition”]
  4. join.php [“requires us to register as well”]

Registering an account, log into it and visit contest.php for more content

Contest.php

Using git-dumper and dumping all the git directories from the website.

From there, try to grep for username & password

from the results,

  1. join.php verifies password in md5 format, which is susceptible to be cracked.
  2. username axel can be confirmed to be an administrator account since admin.php checks for username axel

Furthermore, from git log, the email of axel is exposed

Reflective XSS

  1. Registering with XSS payload as the username
  2. submit a dummy file via contest.php
  3. Host a python server
  4. wait for the XSS payload to reach back out

Using the poc.py

Export the table into user.txt and use awk to print only the password out

Use hashcat to crack it and retrieve the password for user rosa

Use the cracked password to SSH into user rosa

Group adm typically has read access to system logs /var/log, can be useful for privilege escalation due to logs can contain confidential information or credentials.

Checking for files that can be access by group adm

Using find + xargs

notice there are a lot of credentials scattered around and repeating passwords for user axel

Using the same password in the log, SSH into user axel and get user.txt

Notice there’s something interesting: You have mail.

Check for mailbox at /var/mail

/var/mail/axel

only axel is readable, jobert and root was denied for access

From the email, notice that a Gitea which normally runs on port 3000 is currently running at localhost:3000.
Check for other running services

With port forwarding for port 587, 3000 and 25. Access the website on port 3000 which is Gitea.
Port 25 & 587 are open for SMTP Mail Services.

  1. 587 for mail submission (SMTP with authentication)
  2. 25 for mail relay (server-to-server communication)

Port 3000 is running Gitea v1.22.0 which is vulnerable to CVE-2024-6886 (Stored XSS vulnerability)

Steps of exploitation:

  1. setup http listener
  2. create new repo on Gitea
  3. add a file
  4. inject javascript payload in description
  5. trick Jobert into clicking the link
  6. trigger cve

Using password of axel, log in into Gitea

Create a repo to contain the malicious payload

Add a dummy file into the repo and inject JavaScript payload in the Description

In the description, inject the JavaScript payload

Decoding it returns

Using the admin password, we can escalate to root via axel‘s shell
