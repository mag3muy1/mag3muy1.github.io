nmap

fuzz for directories and subdomains

Port 80

  • allows login and registering
  • at the end of page, contains the email for support (support@nocturnal.htb)

Register account as muyi:muyi and logins with it

  • redirects to an upload page

Tried to upload web.php to exploit the file upload vulnerabilties. But the website return the following error:

  • Invalid file type. pdf, doc, docx, xls, xlsx, odt are allowed.

After uploading a test.pdf, notice that it exists in the record

In burpsuite, notice that

  • the website queries using username + file to read the file via view.php
  • using PHPSESSID to authenticate user

Fuzzing more usernames

  • with the session cookie

Therefore, using ffuf and xato-username.txt from seclists

  • from the fuzz above, understand that every response with word count 1170 must be filter

Therefore,

From there, craft the curl payload to see if there’s any file to be downloaded

  • it is clear that user amanda has upload a file named privacy.odt

User

download the file and view it
$ curl "http://nocturnal.htb/view.php?username=amanda&file=privacy.odt" -H "Cookie: PHPSESSID=34k91rcaeqhpn70hvkqg07diss" --output privacy.odt

using the password, amanda:arHkG7HAI68X8s1J. Logged into the website again and notice that user amanda has admin privilege.

Inside the admin panel, we can view all the php files available.

At the end of the page, notice there’s a backup function, using the password discovered for user amanda, backup the entire php files

  • discover nocturnal_database.db
  • download the zip file
